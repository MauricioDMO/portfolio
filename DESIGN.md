# MauricioDMO Portfolio Design System

## Direccion

Este portafolio debe sentirse como una casa personal en internet para mostrar software real, util y construido con criterio tecnico.

La idea central es:

> Software practico para procesos reales.

El sitio no debe parecer un portafolio creativo por presumir creatividad, una landing SaaS generica ni una estetica hacker. Debe comunicar sobriedad, utilidad, experiencia real y cercania.

## Personalidad

- Minimalista oscuro.
- Tecnico sin saturar de terminales.
- Humano sin ser informal de mas.
- Profesional sin sentirse corporativo frio.
- Directo, claro y mantenible.

## Paleta

| Token | Color | Uso |
| --- | --- | --- |
| `--color-bg` | `#0d1210` | Fondo principal |
| `--color-bg-soft` | `#111a15` | Bloques secundarios |
| `--color-card` | `#16211b` | Tarjetas |
| `--color-green` | `#1f5e3b` | Verde bosque principal |
| `--color-green-soft` | `#5faf7b` | Acentos y datos destacados |
| `--color-text` | `#e8eee9` | Texto principal |
| `--color-muted` | `#9baaa0` | Texto secundario |
| `--color-border` | `#26352c` | Bordes finos |

Evitar azul claro, rojo, verde neon, gradientes morados y fondos con particulas.

## Tipografia

- Principal: sans serif moderna de lectura comoda.
- Acento tecnico: monoespaciada solo para etiquetas, tecnologias, metricas y pequenos detalles.
- No usar monoespaciada para parrafos largos.

## Formas

- Bordes pequenos: `4px` a `6px`.
- Bordes finos: `1px solid rgba(255,255,255,0.08)`.
- Tarjetas contenidas, no demasiado redondeadas.
- Sombras muy suaves, priorizando contraste por borde y color.

## Interaccion

- Hover sobrio: borde verde, leve elevacion de `2px`, cambio fino de fondo.
- Transiciones cortas: `160ms` a `220ms`.
- Nada de animaciones exageradas.
- Los CTAs deben ser claros y visibles, no decorativos.

## Contenido

### Tono

Directo, cercano, tecnico y humano.

Evitar frases genericas como:

> Soy un apasionado desarrollador con gran entusiasmo por crear soluciones innovadoras.

Preferir:

> Construyo sistemas web que convierten procesos repetitivos en herramientas utiles, claras y mantenibles.

### Posicionamiento

MauricioDMO es un Full-Stack Developer enfocado en sistemas internos, automatizacion y experiencias web utiles.

El diferenciador principal es mostrar proyectos reales en produccion, entendimiento de negocio, criterio tecnico y enfoque en experiencia de usuario.

## Home

La pagina principal debe tener pocas secciones y mucha intencion:

1. Hero directo.
2. Proyectos destacados.
3. Como trabajo.
4. Stack tecnico.
5. Sobre mi.
6. Contacto.

### Hero

Titulo:

> Software practico para procesos reales.

Texto de apoyo:

> Soy MauricioDMO, Full-Stack Developer. Diseno y construyo sistemas web que ayudan a ordenar operaciones, reducir tareas repetitivas y mejorar la experiencia de quienes usan el software todos los dias.

CTAs:

- Ver casos de estudio.
- Trabajemos juntos.

## Proyectos

Proyectos protagonistas:

1. Sistema de Facturacion Electronica DTE.
2. Sistema de Inventario.
3. Sistema RMA / Garantias.
4. Sitio web de Construfijaciones.

Cada tarjeta debe mostrar problema, impacto, stack y enlace al caso de estudio. El stack no debe reemplazar el contexto del proyecto.

## Casos De Estudio

Cada proyecto debe mantener esta estructura:

1. Resumen.
2. Problema.
3. Solucion.
4. Rol.
5. Funcionalidades principales.
6. Experiencia de usuario.
7. Stack tecnico.
8. Impacto.
9. Aprendizajes.

## Como Trabajo

Proceso base:

1. Entender.
2. Razonar.
3. Plantear.
4. Implementar.
5. Mejorar.

Esta seccion debe mostrar criterio, no solo tecnologias.

## IA

La IA puede mencionarse con sobriedad:

> Uso herramientas de IA como apoyo profesional para documentacion tecnica, depuracion, revision de arquitectura, generacion de pruebas y analisis de soluciones. No como reemplazo del criterio tecnico, sino como una herramienta mas dentro del proceso.

## Reglas De No Diseno

- No barras de progreso de habilidades.
- No porcentajes de tecnologias.
- No tarjetas sin contexto.
- No iconos gigantes de stacks.
- No estetica hacker verde neon.
- No fondos con particulas.
- No textos largos sin jerarquia.
- No decir junior.
